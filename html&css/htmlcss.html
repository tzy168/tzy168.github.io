  • id 属性：

    作用：为 HTML 元素提供唯一的标识符。可用于通过 JavaScript 或 CSS 选择器来引用特定元素。

    示例：<div id="myDiv">这是一个带有唯一 ID 的 div 元素。</div>。

  • class 属性：

    作用：为 HTML 元素分配一个或多个类名。可使用 CSS 选择器为具有相同类名的元素应用相同样式，也可通过 JavaScript 操作具有特定类名的元素。

    示例：<p class="highlight">这是一个带有类名的段落。</p>。

  • style 属性：

    作用：直接在 HTML 元素上应用内联 CSS 样式。可设置元素的颜色、字体大小、边框等样式属性。

    示例：<div style="color: red; background-color: yellow;">带有内联样式的 div。</div>。

  • title 属性：

    作用：为 HTML 元素提供一个标题或说明。当鼠标悬停在元素上时，通常会显示 title 属性的值作为工具提示。

    示例：链接文本。

  • data-* 属性：

    作用：允许开发者在 HTML 元素上存储自定义的数据集。可用于在 JavaScript 中存储和检索与特定元素相关的数据。

    示例：<div data-user-id="123">带有自定义数据属性的 div。</div>。

  • accesskey 属性：

    作用：为 HTML 元素设置快捷键。当用户按下指定的快捷键时，可以快速聚焦或激活该元素。

    示例：（按下 Alt + s 可以快速聚焦这个输入框）。

  • tabindex 属性：

    作用：定义元素在 Tab 键顺序中的位置。可控制用户在使用 Tab 键导航页面时元素的聚焦顺序。

    示例：（这个输入框在 Tab 键顺序中具有较高的优先级）。

CSS 属性 描述
font-family 设置文本的字体。例如：font-family: Arial, sans-serif;
font-size 设置字体大小。
font-weight 设置字体的粗细。可以使用数值（100 - 900）或关键字（normal、bold 等）。例如：font-weight: bold;
font-style 设置字体的样式，如正常（normal）、斜体（italic）等。
text-decoration 添加文本装饰，如下划线（underline）、上划线（overline）、删除线（line-through）等。
color 设置文本颜色。
background-color 设置元素的背景颜色。
width 设置元素的宽度。
height 设置元素的高度。
margin 设置元素的外边距。例如：margin: 10px 20px 10px 20px;（上、右、下、左）或margin: 10px 20px;（上下为 10px，左右为 20px）
padding 设置元素的内边距。用法与margin类似
border 设置元素的边框。可以分别设置边框的宽度、样式（如 solid、dashed 等）和颜色。例如：border: 1px solid black;。
display 设置元素的显示方式，常见的值有block（块级元素）、inline（行内元素）、inline-block（行内块元素）等。
float 设置元素的浮动。可以使元素向左或向右浮动
clear 清除浮动。用于防止元素被浮动元素包围。例如：clear: both;。
position 设置元素的定位方式。常见的值有static（默认，无定位）、relative（相对定位）、absolute（绝对定位）、fixed（固定定位）。例如：position: relative;。
opacity 设置元素的不透明度。取值范围从 0（完全透明）到 1（完全不透明）
cursor 设置鼠标指针在元素上的样式。例如：cursor: pointer;（手形指针）。
overflow 处理内容超出元素尺寸的情况。可以设置为visible（默认，内容可见）、hidden（隐藏超出部分）、scroll（出现滚动条）等。

伪类选择器

伪类选择器用于选择处于特定状态或具有特定行为的元素。伪类选择器以一个冒号（:）开头

  • 基本用法

    • a:hover { color: red; }

    伪类选择器由选择器和冒号（:）组成。例如，选择鼠标悬停在a元素上时的样式。

伪类选择器 描述
:link 选择未被访问过的链接。
:visited 选择已被访问过的链接。
:hover 选择鼠标悬停在其上的元素。
:active 选择正在被激活（如被点击）的元素。
:focus 选择获得焦点的元素，如输入框在被选中时。

:last-child 选择父元素的最后一个子元素。
:nth-child(n) 选择父元素的第 n 个子元素，可以使用具体的数字、表达式或关键字。
:checked 选择被选中的元素，如单选框或复选框。
:disabled 选择被禁用的元素，如不可点击的按钮。

    伪元素选择器由选择器和两个冒号（::）组成。例如，选择p元素的第一行。

css三大特性
  • 层叠性

    • 层叠性是指当多个样式规则应用于同一个元素时，浏览器会根据一定的规则来确定最终应用的样式。 这些规则包括选择器的优先级、样式的来源（如内联样式、内部样式表、外部样式表）以及样式规则的出现顺序等。

  • 继承性

    • 继承性是指某些 CSS 属性可以从父元素传递给子元素。这使得在设置样式时，可以通过为父元素设置样式，让子元素自动继承部分属性，从而减少重复的代码。

  • 优先级

    • 优先级是指在 CSS 中，当多个选择器同时应用于同一个元素时，浏览器会根据选择器的优先级来决定最终应用的样式。优先级的规则是：
    内联样式（inline style） > 内部样式表（internal style sheet） > 外部样式表（external style sheet）。

细说font-size

单位

像素（px）：
定义：最常用的单位之一，表示绝对大小。例如，font-size: 16px;表示字体大小为 16 像素。
特点：在不同设备上显示的大小相对固定，但在高分辨率设备上可能显得较小。
适用场景：适用于需要精确控制字体大小的情况，如设计要求严格的界面或需要与特定图形元素对齐的文本。
百分比（%）：
定义：相对于父元素的字体大小进行设置。例如，如果父元素的字体大小为 16px，子元素设置为font-size: 120%;，则子元素的字体大小为 19.2px（16px * 120%）。
特点：具有相对灵活性，随着父元素字体大小的变化而变化。
适用场景：当需要子元素的字体大小与父元素保持一定比例关系时，可以使用百分比单位。
em：
定义：也是相对于父元素的字体大小进行设置。1em 等于当前元素的字体大小。如果父元素的字体大小为 16px，子元素设置为font-size: 1.2em;，则子元素的字体大小为 19.2px（16px * 1.2）。
特点：与百分比类似，具有相对灵活性，并且在一些情况下更易于计算和调整。
适用场景：常用于响应式设计中，使字体大小能够根据不同的屏幕尺寸和布局进行自适应调整。
rem：
定义：相对于根元素（通常是元素）的字体大小进行设置。例如，如果根元素的字体大小为 16px，其他元素设置为font-size: 1.5rem;，则字体大小为 24px（16px * 1.5）。
特点：提供了一种全局统一的字体大小调整方式，避免了多层嵌套元素中字体大小计算的复杂性。
适用场景：在需要保持整个页面字体大小一致性的情况下非常有用，尤其是在响应式设计中。
vw,vh：
定义：相对于视口的宽度和高度进行设置。例如，font-size: 10vw;表示字体大小为视口宽度的 10%。
特点：相对于视口的尺寸进行设置，具有较好的响应式适配能力。

元素的显示模式
  • 块级元素

    • 独占一行，无论其内容多少，都会在垂直方向上占据整个可用空间。
    可以设置宽度、高度、内边距（padding）和外边距（margin）。
    常见的块级元素有<div><p><h1>-<h6><dul><ol> 等。

  • 行内元素

    • 不会独占一行，只占据其内容所需的空间。
    无法设置宽度和高度，内边距和外边距只对左右有效。
    常见的行内元素有<span><a><img>等。

  • 行内块元素（inline-block）

    • 结合了行内元素和块级元素的特点。
    可以在同一行内显示，同时可以设置宽度、高度、内边距和外边距。
    常见的行内块元素有<input>、<button>等。

  • 弹性盒子（flex）

    • 一种用于创建灵活的布局的技术。
    可以在容器内创建多个弹性子元素，通过设置弹性盒子的属性来控制子元素的布局。
    常见的应用场景包括创建水平导航栏、创建响应式布局等。

  • 网格布局（grid）

    • 可以将页面划分为二维的网格，方便地进行布局。
    可以精确地控制网格的大小、间距和对齐方式。
    适用于复杂的布局需求，尤其是需要多行多列的布局。

  • 修改元素的显示模式

    • 可以通过修改元素的display属性来改变元素的显示模式:block、inline、inline-block、flex、gird

    • 浮动的特点
      1. 脱离文档流

        2. 浮动元素会脱离正常的文档流，这意味着它不再占据文档流中的空间，其他元素会无视它的存在进行布局。
        例如，如果一个浮动元素在一个没有设置浮动的块级元素内部，这个块级元素的高度将不会包含浮动元素，除非采取特殊的措施来清除浮动。

      2. 依次排列

        3. 多个浮动元素会按照它们在 HTML 中的顺序依次排列，如果空间不足，浮动元素会自动换行。

    • 影响
      1. 父元素高度塌陷

        2. 如果一个父元素包含浮动元素，而这个父元素没有设置高度，那么它的高度将变为0，导致父元素无法显示内容。

      2. 文本环绕

        3. 如果一个元素设置了浮动，那么它会脱离文档流，导致其他元素会环绕它。

      3. 相邻元素布局改变

        4. 相邻的浮动元素会相互影响布局。如果两个浮动元素宽度之和超过容器宽度，后面的元素会自动换行。
        未浮动的块级元素在遇到浮动元素时，其布局可能会受到干扰，可能会出现重叠或奇怪的间距问题。

      4. 脱离文档流

        5. 浮动元素会脱离文档流，这意味着它们不再占据文档流中的空间，其他元素会无视它们的存在进行布局。

      5. 按指定方向排列

        6. 多个浮动元素会按照它们在 HTML 中的顺序依次排列，如果空间不足，浮动元素会自动换行。

    • 清除影响
      1. 使用clearfix类

        2. 使用clearfix类可以清除浮动元素对父元素的影响。 .clearfix { clear: both; }

      2. 使用overflow属性

        3. 将父元素的overflow属性设置为auto、hidden或scroll，可以清除浮动元素对父元素的影响。

      绝对定位

      • 固定定位的元素是相对于浏览器窗口进行定位。

    • sticky

      • 初始状态：在页面滚动时，粘性定位的元素在其原始位置上表现得像相对定位的元素，随着页面一起滚动。
      触发条件：当满足特定的滚动条件时，粘性定位的元素会切换为固定定位，停留在页面的特定位置。
      依赖关系：粘性定位的元素通常依赖于其父元素的边界来确定其位置。例如，可以设置一个元素在其父元素的顶部或底部保持粘性。

    1. 语义化标签：
      2. HTML5 引入了一系列语义化标签, 如 header（页眉）、nav（导航栏）、section（章节）、article（文章）、aside（侧边栏）和footer（页脚）等。 这些标签使网页的结构更加清晰，便于开发者编写易于理解和维护的代码，同时也有利于搜索引擎优化（SEO）。
    2. 多媒体支持
      3. HTML5 提供了原生的多媒体支持，无需依赖第三方插件（如 Flash）即可播放音频和视频。
      <audio>和<video>标签允许开发者轻松地在网页中嵌入音频和视频文件，并提供了一系列控制属性，如播放、暂停、音量调节等。

      4. HTML5 引入了 <canvas> 元素，允许开发者通过 JavaScript 绘制图形和动画。
      开发者可以使用 JavaScript 来操作 <canvas> 元素，实现各种绘图效果，如绘制图形、绘制图表、绘制动画等。
    4. 本地存储：
      5. HTML5 引入了本地存储（Local Storage）和会话存储（Session Storage），允许网页在用户的浏览器中存储数据。
      本地存储的数据在浏览器关闭后仍然存在，而会话存储的数据在浏览器关闭时会被清除。
      // 存储数据到本地存储
      localStorage.setItem('key', 'value');
      // 从本地存储获取数据
      var value = localStorage.getItem('key');
      // 移除本地存储中的数据
      localStorage.removeItem('key');
    5. 表单增强：
      6. HTML5 对表单进行了大量的增强， 包括新的输入类型（如email、url、number、date等）、表单验证属性（如required、pattern等）和自动完成功能。 这些增强使得表单的创建更加容易，同时也提高了用户体验和数据的准确性。

    1. 浏览器前缀
      2. 浏览器前缀是为了兼容不同浏览器的CSS属性和值。
      例如，在CSS3中使用了新的属性transform，但是不同的浏览器对这个属性的支持程度不同，
      所以需要在不同的浏览器中添加相应的前缀，以确保在不同的浏览器中都能正常显示。
      例如，在Chrome浏览器中使用transform属性，需要添加前缀-webkit-，
      而在Firefox浏览器中使用transform属性，需要添加前缀-moz-。
      .box {
      -webkit-transform: rotate(45deg);
      -moz-transform: rotate(45deg);
      transform: rotate(45deg);
      }
    2. 浏览器版本
      3. 不同的浏览器版本可能对CSS3属性的支持程度不同，
      因此在使用CSS3属性时，需要注意浏览器版本的兼容性。
      例如，在CSS3中使用了新的属性transform，但是在某些旧版本的浏览器中可能不支持，
      所以需要在使用时进行兼容性处理。
      例如，在使用transform属性时，可以使用CSS3的@supports规则来判断浏览器是否支持该属性，
      如果支持，则使用该属性，否则使用其他属性或不使用该属性。
      @supports (transform: rotate(45deg)) {
      .box {
      transform: rotate(45deg);
      }
      }

      -o-：用于Opera浏览器
      -ms-：用于IE浏览器
